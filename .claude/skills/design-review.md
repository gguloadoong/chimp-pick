# 디자인 리뷰 🎨

## 진행 절차

### 1. 디자인 발표
**박도파민 (Designer)**:
- 리뷰 대상 화면/컴포넌트 설명
- 디자인 의도 및 원칙 설명
- 인터랙션/애니메이션 가이드

### 2. 피드백 수집

**김가즈아 (PM)**: 프로덕트 관점
- 유저 스토리와 부합하는가?
- 비즈니스 목표에 기여하는가?

**이떡상 (FE)**: 구현 관점
- 기술적 구현 가능성
- 성능 영향 평가
- 구현 공수 산정

**한물타기 (Strategist)**: 그로스 관점
- 바이럴 요소 충분한가?
- 공유하고 싶은 화면인가?
- 도파민 테스트 통과?

**장손절 (QA)**: 품질 관점
- 엣지케이스 UI (빈 상태, 에러, 로딩)
- 접근성 (색맹, 폰트 크기)
- 반응형 (다양한 디바이스)

### 3. 수정 사항 합의
- 반영할 피드백 정리
- 수정 우선순위 결정
- 디자인 스펙 업데이트 여부 결정

### 4. 기록
- `.project/design-spec.md` 업데이트 (변경 시)

## 사용법
\`\`\`
/design-review "메인 게임 화면"
/design-review "결과 애니메이션"
\`\`\`

## 산출물
- 피드백 요약
- 수정 액션 아이템
- 디자인 스펙 업데이트
