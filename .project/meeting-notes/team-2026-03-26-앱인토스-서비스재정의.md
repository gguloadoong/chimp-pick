## 안건 1. 앱인토스(App Store) 심사 통과 위험 요소

### 조사 결과

| 위험 요소 | 등급 | 핵심 문제 |
|---|---|---|
| IAP + 배당 연결 구조 | **HIGH** | 실제 돈 → 바나나코인 → 배당 흐름 = 시뮬레이션 도박 |
| 5.3 도박 조항 | MEDIUM | 배당률 메커니즘이 Apple 가이드라인 경계선 |
| 바이너리 옵션 유사성 | MEDIUM | UP/DOWN 2진 예측 + 배당 = 3.2.2(viii) 지적 가능 |
| 한국 GRAC 등급분류 | MEDIUM | 사행성 게임 분류 시 등급분류 자체가 불가 |
| 금융 정보 라이센스 | LOW | 게임 포지셔닝 시 면제 가능 |

### 핵심 리스크
바나나코인 인앱 구매(IAP) + 베팅 배당 구조가 연결되면 **"실제 돈 → 가상 화폐 → 배당"** 흐름이 성립하여 도박 앱으로 분류될 위험이 있음.

- IAP 바나나코인 → **스킨/칭호/코스메틱만** 구매 가능
- 베팅용 바나나코인 → 일일 무료 지급/연승 보상/초대 보상만으로 획득
- 앱 카테고리: **Games > Simulation** 제출
- 앱 설명에서 "베팅", "배당", "수익" 용어 제거 → "예측 포인트", "정확도 보상"으로 교체

- Apple Developer 계정: **법인 명의** 등록 필수
- GRAC: 출시 전 사전 질의 → "전체이용가" 등급 목표

## 안건 2. 서비스 본질 재정의 (상위 기획)

### 재정의

> **"주식/코인을 소재로 한 군중 심리 예측 게임 — 맞추는 게 목적이 아니라, 다수파/소수파 심리 분포 자체가 콘텐츠다."**

시장을 분석하는 서비스가 아님. 사람들이 뭘 믿는지를 구경하고, 거기에 베팅하는 사회적 놀이터.

### 핵심 가치 3가지

1. **병맛 몰입** — 침팬지 세계관, "분석"이 아닌 "감"으로 찍는 게 정당화되는 공간
2. **군중 신호** — 집계된 예측 분포가 유일한 정보 레이어 ("80%가 UP 찍었는데 DOWN 갈 용기?")
3. **소셜 배틀** — "너 또 틀렸잖아"가 발생하는 구조 → 리텐션의 핵심

### 병맛+정보 균형 원칙

| 구분 | 원칙 |
|------|------|
| UI 비율 | 병맛 70% / 정보 30% |
| 정보의 정체 | 시장 데이터 아닌 **군중 데이터** (투표 분포, 소수파 비율, 연승) |
| 시장 데이터 | 현재가 한 줄만 메인 노출. 차트는 "더보기" 뒤에 숨김 |
| 톤 검증 기준 | "이걸 증권사 앱에서 볼 수 있나?" YES → 리디자인 대상 |
| 텍스트 | "매수 신호" → "바나나 올인 각", "하락 전망" → "침팬지 도망감" |

### 감정 곡선 설계

```
참여: "뭐야 이 앱 ㅋㅋ" (진입 장벽 제로, B급 인상)
예측: "다들 UP인데 나만 DOWN...?" (사회적 압력 + 소수파 역배당)
대기: 실시간 분포 변동 보며 긴장감 유지
결과: 맞추면 바나나 폭죽, 틀리면 "역시 침팬지" 자조 밈
      → 둘 다 공유하고 싶게 설계 (승리=자랑, 패배=공유)
```

### 수익 모델 방향 (앱 심사 안전)

- IAP: 코스메틱 전용 (스킨, 승패 이펙트, 배지)
- 배틀패스: 시즌별 미션 + 보상 트랙 (무과금 플레이 보장)
- **절대 금지**: BC 현금 환전, 실제 금전 보상, 투자 조언

## 안건 3. 채도 높은 픽셀 디자인 리서치

### 팔레트 업그레이드: Sweetie 16 → ENDESGA-32

| 역할 | 현재 (Sweetie 16) | 권장 (ENDESGA-32) | 변화 |
|------|------------------|-------------------|------|
| UP 초록 | `#a7f070` | `#63c74d` | 채도 +30% |
| DOWN 빨강 | `#b13e53` (탁함) | `#e43b44` (순수 레드) | 채도 +50% |
| 브랜드 옐로 | `#ffcd75` | `#feae34` (앰버) | 따뜻함+ |
| CTA 강조 | 없음 | `#ff0044` (마젠타레드) | 신규 |
| 배경 | `#1a1c2c` | `#1b1b1b` (니어블랙) | 더 순수한 다크 |
| 텍스트 | 흰색 계열 | `#fee761` (레트로 옐로) | 픽셀 감성+ |

PICO-8 대안도 검토: `#ff004d` DOWN, `#00e436` UP (더 네온 계열)

### 타이포그래피

- **Galmuri11** (npm: `galmuri`) — Nintendo DS 영감, 한글 완벽 지원, SIL OFL 무료
  - Galmuri14: 현재가/대형 숫자
  - Galmuri11 Bold: UP/DOWN 버튼
  - Galmuri9: 배지/태그
- **NeoDungGeunMo** — 둥근 픽셀 감성, CDN으로 @font-face 적용

### 핵심 CSS 패턴

```css
/* 픽셀 버튼 3D 효과 */
box-shadow: inset -4px -4px 0 rgba(0,0,0,0.3), inset 4px 4px 0 rgba(255,255,255,0.2);

/* 픽셀 단계 전환 (스무딩 제거) */
transition: color 0.3s steps(6);

/* 픽셀 셰이크 애니메이션 */
animation: pixel-shake 0.5s steps(1) forwards;
```

### 레퍼런스 라이브러리

- **Pixelact UI** (shadcn/ui 기반 픽셀 컴포넌트, Next.js 호환): `npx shadcn@latest add https://pixelactui.com/r/button.json`
- Lospec: ENDESGA-32, Sweetie 16, PICO-8
