## 1. 김가즈아 (PM) — 프로젝트 개요

### 비전
"누구나 30초 만에 주식/코인 등락을 예측하는 B급 밈 배틀 게임"

### 핵심 가치
- 진입 장벽 제로: 종목 고르고 UP/DOWN 누르면 끝
- 리스크 제로: 바나나코인으로 베팅, 실제 돈 아님
- 재미 극대화: B급 밈 감성, 침팬지 캐릭터, 게이미피케이션

### MVP 스코프 (P0)
- 인증: 카카오 로그인, 게스트 모드, JWT, 닉네임
- 게임 코어: 종목 선택, UP/DOWN 예측, 실시간 차트, 결과 판정, 바나나코인
- 시세: Upbit(코인), KIS(주식) API
- 인프라: scaffolding, DB, CI/CD

### 스프린트 일정 (13주)
| 스프린트 | 기간 | 목표 | 마일스톤 |
|---------|------|------|---------|
| Sprint 0 | 3/23~3/29 | 기반 세팅 | - |
| Sprint 1-2 | 3/30~4/26 | 코어 루프 | 내부 알파 |
| Sprint 3-4 | 4/27~5/24 | 소셜+랭킹 | 클로즈드 베타 (100명) |
| Sprint 5-6 | 5/25~6/21 | 그로스+최적화 | 퍼블릭 런칭 |

### 팀 당부
- 스펙 먼저, 코드는 나중에
- 롤백 가능한 설계
- 숫자로 대화
- 15분 스탠드업
- 역할 존중

## 2. 박도파민 (Designer) — 디자인 방향

### 디자인 원칙
1. 도파민 퍼스트: 모든 인터랙션에 즉각적 시각 피드백
2. 엄지 영역 최적화: 핵심 액션 하단 40%
3. B급이되 조잡하지 않게
4. 다크 모드 기본

### 디자인 토큰
- 컬러: #FFB800(바나나옐로), #00D68F(상승), #FF4757(하락), #1A1A2E(배경), #16213E(카드)
- 폰트: Pretendard(본문), 넥슨Lv1고딕(밈), JetBrains Mono(숫자)
- 스페이싱: 4px 그리드 (4, 8, 12, 16, 24, 32, 48)
- 네온 글로우 이펙트

### B급 톤앤매너
- 의도적 과잉: "정성스러운 B급"
- 로딩: "침팬지가 바나나 가져오는 중..."
- 에러: "우끼끼... 뭔가 잘못됐다 침팬지야"
- 모든 빈 상태에 침팬지 캐릭터 배치

### 캐릭터 진화 5단계
1. 아기원숭이 (가입)
2. 침팬지 (10회+ 예측, 적중률 40%+)
3. 고릴라 (50회+, 적중률 55%+ 또는 연승 10회)
4. 킹콩 (200회+, 적중률 60%+ 또는 연승 25회)
5. 전설의침팬지 (시즌 상위 1% 또는 특수 업적)

### 애니메이션 전략
- 60fps 사수, transform + opacity만 사용
- 적중: 폭죽 + 바나나 비 (2초)
- 실패: 바나나 껍질 미끄러짐 (0.8초)
- 진화: 풀스크린 변신 (3초)
- `prefers-reduced-motion` 필수 적용

### Sprint 0 디자인 딜리버리
- 디자인 토큰 Figma Variables 완성
- 기본 컴포넌트 제작
- 침팬지 캐릭터 1~2단계 일러스트
- 메인 게임 화면 목업
- Lottie 애니메이션 1차 (적중/실패)
- 인터랙티브 프로토타입

## 3. 이떡상 (FE) — 프론트엔드 기술 스택

### 기술 스택
| 기술 | 선택 이유 |
|------|----------|
| Next.js 15 App Router | RSC로 번들 축소, Streaming SSR |
| TypeScript strict | any 쓰면 PR 리젝 |
| Tailwind CSS | 유지보수 비용 최소화 |
| Zustand + React Query | 클라이언트/서버 상태 분리 |
| Lightweight Charts | 실시간 캔들차트 최고 성능 |
| Framer Motion + Lottie | 인터랙션 + 캐릭터 애니 분리 |
| Socket.IO | 재연결, 폴백 기본 제공 |
| Vitest + Playwright | 빠른 유닛 + 안정 E2E |

### 실시간 처리 전략
- 가격 데이터 100ms 스로틀링
- 배치 업데이트 (`unstable_batchedUpdates`)
- 차트 데이터 `useMemo`, 이벤트 핸들러 `useCallback`
- 랭킹 리스트 가상화 (`@tanstack/virtual`)

### 성능 목표
- 번들 < 200KB (initial), FCP 1.5s, LCP 2.5s, CLS < 0.1
- Lighthouse 90+
- 200KB 초과 PR 리젝

### Sprint 0 FE 딜리버리
- Next.js 15 scaffolding + ESLint/Prettier
- Tailwind 디자인 토큰 설정
- 공용 컴포넌트 (Button, Input, Modal, Toast)
- OpenAPI 타입 자동 생성 파이프라인
- WebSocket 훅 구조 (`useWebSocket`)
- CI (Vitest, Playwright, bundle-analyzer)

## 4. 최풀매수 (BE) — 백엔드 기술 스택

### 기술 스택
| 기술 | 선택 이유 |
|------|----------|
| NestJS 10 | 구조 강제, DI, 모듈화 |
| Prisma | SQL injection 원천 차단, 타입 자동 생성 |
| PostgreSQL 16 | ACID, row-level lock |
| Redis 7 | 캐시 + Pub/Sub |
| TimescaleDB | 시계열 파티셔닝, 자동 압축 |
| BullMQ | 비동기 큐, 재시도, 데드레터 |
| JWT + HttpOnly Cookie | XSS 방어 |

### 7대 모듈 구조
- auth: 인증/인가
- user: 프로필, 진화 단계
- game: 예측 생성/판정 (핵심)
- price: 외부 시세 수집/캐싱
- ranking: Redis Sorted Set 실시간 랭킹
- reward: 바나나코인 증감 전용 (직접 DB 수정 금지)
- notification: 인앱 알림

### 실시간 시세 파이프라인
```
[Upbit WS / KIS API] → PriceCollector → Redis Cache + Pub/Sub → Socket.IO → Client
                                ↓
                         TimescaleDB → CandleAggregator (BullMQ)
```

### 장애 대비
- Upbit 다운 → Binance 자동 페일오버
- 시세 지연 3초 → 해당 종목 예측 차단
- 전체 다운 → 캐시 제공 + "지연 데이터" 표시

### 예측 판정 로직
- entryPrice: 예측 생성 시 Redis에서 즉시 고정
- BullMQ 워커가 만료 시점에 판정
- PostgreSQL 트랜잭션: 예측 상태 + 코인 증감 + UserStats 원자적 처리
- 시세 없으면 10초 후 재시도, 3회 실패 시 취소+환불

### DB 핵심 테이블
- User (cuid, nickname unique, avatarLevel, bananaCoins)
- Prediction (entryPrice/exitPrice Decimal, result ENUM)
- UserStats (denormalized 집계, 판정 시 동시 업데이트)
- Transaction (감사 로그, balanceAfter 포함)
- PriceCandle (TimescaleDB, 복합 PK)

### Sprint 0 BE 딜리버리
- NestJS scaffolding + ESLint/Prettier
- Docker Compose (PostgreSQL, Redis, TimescaleDB)
- Prisma 스키마 v1 + 마이그레이션
- auth 모듈 완성 (회원가입, 로그인, 토큰)
- Swagger API 문서 자동화
- GitHub Actions CI
- 테스트 DB 시딩 스크립트

## 5. 한물타기 (Strategist) — 그로스 전략

### 타겟 유저 4대 페르소나
| 페르소나 | 연령 | 핵심 동기 | 역할 |
|---------|------|----------|------|
| 밈 투자러 | 25-32 | 자기표현, 소셜 인정 | K-factor 엔진 |
| 점심시간 겜러 | 22-28 | 재미, 시간 때우기 | DAU 볼륨 |
| 랭킹 헌터 | 20-35 | 경쟁, 성취 | D7/D30 리텐션 |
| 관전러 | 18-30 | 엔터테인먼트 | MAU, 광고 소비 |

### North Star Metric
일일 예측 수 (Daily Predictions)

### 핵심 KPI
| 메트릭 | 목표 |
|--------|------|
| 가입→첫예측 전환율 | 70% |
| D1 리텐션 | 40% |
| D7 리텐션 | 20% |
| D30 리텐션 | 10% |
| K-factor | 0.3 |
| 예측/유저/일 | 5회+ |

### 런칭 3-Phase
1. 프리런칭 (5/24~6/6): 밈 시딩, 대기자 3,000명, 얼리어답터 100명 베타
2. 소프트런칭 (6/7~6/20): 초대 코드 제한, K-factor 측정, A/B 테스트
3. 퍼블릭 런칭 (6/21): DAU 10K 목표

### 바이럴 핵심 인사이트
- 실패 콘텐츠가 성공 콘텐츠보다 공유율 1.4배 (schadenfreude 활용)
- 공유 트리거 5가지: 연승, 연패, 랭킹 진입, 친구 대결, 레어 칭호

### 리텐션 3 레이어
1. 습관 형성 (D1-D7): 스트릭, 일일 보상, loss aversion
2. 진행감 (D7-D30): 침팬지 진화, 칭호, 시즌제
3. 소셜 잠금 (D30+): 친구 대결, 길드, 랭킹 경쟁

### Sprint 0 전략 딜리버리
- 이벤트 택소노미 정의서 (P0)
- 온보딩 퍼널 설계 (P0)
- 공유 카드 기획 (P1)
- 바나나코인 이코노미 v1 (P1)

## 6. 장손절 (QA) — 품질 전략

### 테스트 피라미드
- Unit (Vitest/Jest): 비즈니스 로직 80%+ 커버리지
- Integration (Supertest): 모든 API + DB 연동
- E2E (Playwright): 5대 핵심 플로우만

### 5대 E2E 핵심 플로우
1. 회원가입 → 첫 예측
2. 예측 → 결과 → 보상 (코인 정합성)
3. 랭킹 확인
4. 프로필 및 전적 확인
5. 공유 카드 생성

### 릴리즈 품질 게이트 (Blocking)
- P0/P1 버그: 0건
- E2E 5개 플로우: 전체 통과
- Unit 커버리지: 80%+
- API p95 < 200ms
- WebSocket < 100ms

### 버그 심각도
| 등급 | 정의 | 대응 |
|------|------|------|
| P0 | 금액 오류, 서버 다운, 데이터 유실 | 즉시 핫픽스 |
| P1 | 핵심 기능 장애 | 당일 수정 |
| P2 | UI 깨짐, 비핵심 오류 | 다음 스프린트 |
| P3 | 오타, 미세 이슈 | 백로그 |

### 보안 테스트 필수 항목
- SQL Injection, XSS, JWT 조작, IDOR, Rate Limiting
- 음수 베팅, 마감 시간 우회

### Sprint 0 QA 딜리버리
- QA 전용 테스트 환경 구성
- E2E 시나리오 상세 문서화
- 보안 체크리스트 완성
- `data-testid` 컨벤션 확정
- CI 테스트 파이프라인 연결
