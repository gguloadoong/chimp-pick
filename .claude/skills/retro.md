# 스프린트 회고 🔄

## 진행 절차

### 1. 데이터 수집
**김가즈아 (PM)**:
- 스프린트 목표 달성률
- 완료/미완료 항목 정리
- 주요 메트릭 변화

### 2. 팀 회고 (각 에이전트)
각 에이전트가 다음 3가지를 공유:

\`\`\`
🟢 잘한 점 (Keep): 계속 유지할 것
🟡 개선할 점 (Problem): 개선이 필요한 것
🔵 시도할 점 (Try): 다음에 해볼 것
\`\`\`

### 3. 투표 & 우선순위
- 가장 임팩트 큰 개선 포인트 3개 선정
- 각 개선 포인트에 담당자 + 구체적 액션 지정

### 4. 액션 아이템
- 다음 스프린트에 반영할 개선 사항 확정
- `.project/meeting-notes/retro-sprint{N}-{날짜}.md`에 저장

## 사용법
\`\`\`
/retro          # 현재 스프린트 회고
/retro 2        # Sprint 2 회고
\`\`\`

## 산출물
- 회고 기록 (회의록)
- 개선 액션 아이템 (다음 스프린트 반영)
