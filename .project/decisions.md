## 결정 기록

### DEC-001: 프레임워크 선택 (2026-03-20)
- **결정**: FE: Next.js 15 / BE: NestJS 10
- **대안**: FE: Remix, Vite+React / BE: Express, Fastify
- **이유**:
  - Next.js: App Router + RSC로 초기 로딩 최적화, Vercel 배포 편의성
  - NestJS: 모듈화 구조로 팀 협업 용이, TypeScript 네이티브
- **결정자**: 이떡상 (FE) + 최풀매수 (BE)
- **상태**: 확정

### DEC-002: 실시간 통신 방식 (2026-03-20)
- **결정**: Socket.IO (WebSocket + polling fallback)
- **대안**: 순수 WebSocket, SSE, gRPC-Web
- **이유**: 자동 재연결, room/namespace, fallback 지원. 업비트 시절 검증됨
- **결정자**: 최풀매수 (BE)
- **상태**: 확정

### DEC-003: 배팅 배율 1.8x (2026-03-20)
- **결정**: 적중 시 베팅액 × 1.8 회수 (순이익 0.8x)
- **대안**: 2.0x (순이익 1.0x), 1.5x (순이익 0.5x)
- **이유**:
  - 2.0x면 50% 승률로도 손익분기 → 인플레이션 위험
  - 1.8x면 56% 이상 승률이어야 수익 → 적절한 난이도 + 디플레이션
- **결정자**: 한물타기 (Strategist) + 김가즈아 (PM)
- **상태**: 확정 (런칭 후 데이터 보고 조정 가능)

### DEC-004: 모바일 퍼스트 PWA (2026-03-20)
- **결정**: 네이티브 앱 대신 PWA로 시작
- **대안**: React Native, Flutter
- **이유**:

  - 이후 수요 확인 후 네이티브 전환 검토
- **결정자**: 김가즈아 (PM)
- **상태**: 확정
