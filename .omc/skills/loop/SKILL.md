# 자율 실행 루프 🔄

## 개요
에이전트가 백로그 항목을 자율적으로 처리하는 실행 모드입니다.
CEO(사용자)의 승인이 필요한 결정은 `/request-to-ceo`로 에스컬레이션합니다.

## 진행 절차

### 1. 작업 선택
- `.project/backlog.md`에서 우선순위 순으로 다음 작업 선택
- 의존성 체크 (선행 작업 완료 여부)

### 2. 작업 실행
- 해당 에이전트가 작업 수행
- 필요 시 다른 에이전트 협업 요청

### 3. 검증
- **장손절 (QA)**: 결과물 품질 검증
- 테스트 통과 여부 확인

### 4. 완료 & 다음
- 백로그 항목 완료 처리
- 다음 항목으로 자동 진행
- 블로커 발생 시 CEO에게 보고

### 5. 종료 조건
- 모든 스프린트 백로그 완료
- CEO가 중단 명령
- 해결 불가 블로커 발생

## 사용법
\`\`\`
/loop                    # 전체 팀 자율 실행
/loop fe                 # FE(이떡상)만 자율 실행
/loop fe,be              # FE + BE 자율 실행
/loop --sprint 2         # Sprint 2 백로그 대상
\`\`\`

## 주의사항
- 큰 의사결정은 반드시 `/request-to-ceo`로 확인
- 작업 진행 상황을 주기적으로 보고
- 에이전트 간 충돌 시 PM(김가즈아)이 조율
