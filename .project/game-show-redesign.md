# 침팬지픽 "30초 게임쇼" 화면 재설계

> 작성: 김가즈아 (PM) | 2026-03-25
> 핵심 전환: 트레이딩 앱 -> 밤의 게임쇼 (Jackbox + Kahoot! + B급 밈)

---

## 0. 설계 철학

**"이건 차트 앱이 아니라, 30초짜리 TV 게임쇼다."**

| 기존 (트레이딩 앱) | 전환 (게임쇼) |
|---|---|
| 캔들차트가 주인공 | 질문이 주인공 |
| 정보 밀도 높음 | 감정 밀도 높음 |
| 분석하는 UX | 직감으로 찍는 UX |
| 조용한 결과 표시 | 빵! 터지는 결과 공개 |
| 설정이 많음 | 선택지가 2개뿐 |

레퍼런스: Kahoot!(긴장감 있는 카운트다운), Jackbox(한 화면에 모두 집중), 복불복 게임쇼(결과 공개의 쾌감)

---

## 1. 카테고리별 색상 테마

각 카테고리가 등장할 때 화면 전체의 분위기가 바뀐다. TV 게임쇼의 "코너 전환" 느낌.

```
카테고리      | 메인 컬러  | 배경 그라데이션         | 아이콘  | 무드
-------------|-----------|----------------------|--------|--------
시세 예측     | #FFD700   | 검정 -> 짙은 금색      | 📈     | 긴장/진지
재미 예측     | #FF6B9D   | 짙은 보라 -> 핑크      | 🎲     | 장난/가벼움
상식 퀴즈     | #4ECDC4   | 짙은 청록 -> 민트      | 🧠     | 도전/자신감
스포츠       | #45B7D1   | 짙은 네이비 -> 하늘    | ⚽     | 열정/흥분
트렌드       | #96CEB4   | 짙은 올리브 -> 연두    | 📰     | 생각/토론
```

구현: 카테고리가 바뀔 때 `body`의 CSS 변수를 전환. `transition: background 0.5s ease`.
효과: "아, 이번엔 상식이네" — 색상만으로 카테고리를 인지.

## 2. 화면 1: 로그인/랜딩

### 설계 의도
- 3초 안에 "이건 예측 게임이다"를 전달
- CTA 1개. 고민할 게 없어야 함
- TV 게임쇼 오프닝 느낌: 어둡다가 조명이 켜지는 연출

### ASCII 와이어프레임

```
+------------------------------------------+
|                                          |
|           (어두운 배경, 별 파티클)          |
|                                          |
|              🦍                           |
|          (침팬지 캐릭터)                    |
|        (마이크 들고 있는 포즈)              |
|                                          |
|     ━━━━━━━━━━━━━━━━━━━━━━━━━━           |
|                                          |
|        침 팬 지 픽                         |
|      CHIMP PICK                          |
|                                          |
|    " UP일까 DOWN일까,                     |
|      30초 안에 찍어! "                     |
|                                          |
|     ━━━━━━━━━━━━━━━━━━━━━━━━━━           |
|                                          |
|   지금 142명이 게임 중 🔴 LIVE             |
|                                          |
|   +------------------------------------+ |
|   |                                    | |
|   |    🍌  게 임  시 작  하 기           | |
|   |                                    | |
|   +------------------------------------+ |
|          (크고 노란 버튼, 펄스 애니메이션)   |
|                                          |
|     게스트로 3판 무료 체험                   |
|     카카오 | 구글 | 애플                    |
|                                          |
+------------------------------------------+
```

### 세부 설계

**헤드라인 카피**: `"UP일까 DOWN일까, 30초 안에 찍어!"`
- 대안 1: `"찍으면 맞는 게임, 침팬지픽"`
- 대안 2: `"주식도 상식도, 직감 하나로 승부"`

**레이아웃**:
- 상단 60%: 침팬지 캐릭터 + 로고 + 헤드카피 (풀스크린 비주얼)
- 하단 40%: CTA + 소셜 로그인 옵션

**연출**:
- 페이지 로드 시 0.5초 어둠 -> 스포트라이트 ON 효과 (opacity 0 -> 1)
- 침팬지 캐릭터가 마이크 들고 "가즈아!" 포즈
- CTA 버튼에 `pulse` 애니메이션 (2초 주기)
- "지금 N명이 게임 중" — 실시간 접속자 수 (없으면 서버 접속 로그 기반 근사치)

**게스트 모드**:
- CTA 클릭 즉시 게임 진입 (로그인 없이)
- 3판 후 "기록을 저장하려면 가입하세요" 모달
- 소셜 로그인은 CTA 아래에 작게 배치 (게스트 플로우 방해 X)

### 세부 설계

**헤더 (최소화)**:
- 왼쪽: 카테고리 뱃지 (이모지 + 이름, 카테고리 색상)
- 오른쪽: 바나나코인 잔고 + 누적 점수 (작게)
- 침팬지 캐릭터 / 로고 / 설정은 헤더에서 제거 — 게임 중에는 게임만

**질문 카드**:
- 화면 중앙, 가장 큰 폰트 (24px+)
- 카테고리 색상의 좌측 보더 또는 배경 그라데이션
- description은 질문 아래 작은 글씨
- 카드 등장 시 `scale(0.9) -> scale(1)` + `opacity` 트랜지션

**카운트다운 타이머**:
- 원형 프로그레스 (SVG circle, stroke-dashoffset 애니메이션)
- 30초 -> 10초: 카테고리 색상
- 10초 -> 5초: 주황색 + 약한 펄스
- 5초 -> 0초: 빨간색 + 강한 펄스 + 진동(haptic)
- 가운데 숫자 크게 (48px 모노스페이스)

**A/B 선택 버튼**:
- 화면 하단 고정 (thumb zone)
- 두 버튼이 동일한 크기로 나란히
- 누르면 즉시 시각 피드백: 선택된 쪽 확대 + 체크마크, 미선택 쪽 흐림
- 한 번 선택하면 변경 불가 (게임쇼 긴장감)

**참여자 수**:
- 하단에 "47명 참여 중" — 사회적 증거
- 이 숫자는 WebSocket으로 실시간 갱신

## 4. 소수파 비율: 선택 후 공개 (설계 근거)

### 결정: 선택 후 공개

### 근거

| 선택지 | 장점 | 단점 |
|--------|------|------|
| **선택 전 공개** | 정보 기반 판단 가능, "전략적" 느낌 | 쏠림 현상 (다수파로 몰림), 역배 보너스 무력화, 자기 생각 대신 남 따라감 |
| **선택 후 공개** | 독립적 판단 유도, 역배 보너스 의미 있음, "내가 소수파네!" 서프라이즈 | 정보 없이 찍는 느낌 |

**핵심 논리**: 게임쇼에서 관객 투표 결과를 먼저 보여주면 재미가 없다.
"백만장자 쇼"에서도 관객 투표는 "도움 요청" 후에만 공개한다.
선택 전 공개하면 62% 쪽으로 몰리고, 역배 보너스 메커니즘이 의미를 잃는다.

**UX 흐름**:
1. 질문 읽기 -> 직감으로 선택
2. 선택 즉시 -> "소수파: B 38% / 역배 보너스 x1.6" 공개
3. "오, 내가 소수파네?" 또는 "역시 다수파지" -> 감정 발생
4. 카운트다운 끝날 때까지 다른 사람들의 선택이 실시간으로 바뀌는 바 차트

### 세부 설계

**결과 공개 연출 (2초 서스펜스)**:
1. 카운트다운 0 -> 화면 어두워짐 (0.3초)
2. 드럼롤 사운드 + 화면 중앙 "?" 깜빡임 (1초)
3. 정답 공개 -> 맞으면 초록 플래시 + 환호 사운드 / 틀리면 빨강 플래시 + 꽝 사운드 (0.5초)
4. 점수/보상 표시 (즉시)

**카피 — 맞았을 때** (랜덤 순환):
- "역시 천재 침팬지!"
- "가즈아아아! 적중!"
- "침팬지 직감은 과학이다"
- "바나나 수확 완료!"
- "월급보다 정확한 예측"

**카피 — 틀렸을 때** (랜덤 순환):
- "바나나 미끄러졌다"
- "침팬지도 가끔 틀린다..."
- "다음 판에 복수하자"
- "바나나 증발... 다시 가즈아!"
- "틀려도 괜찮아, 원래 확률은 50%야"

**점수 표시**:
- 기본 점수: 맞으면 +100, 틀리면 0
- 소수파 보너스: 맞음 + 소수파였으면 점수 x 역배 배수 (예: x1.6)
- 연승 보너스: 3연승부터 +20% 추가
- 바나나코인 변동: 맞으면 +BC, 틀리면 -BC

**다음 라운드 전환**:
- 3초 카운트다운 프로그레스 바
- 자동 전환 (터치하면 즉시 넘어감)
- 공유 버튼은 전환 전에 누를 수 있도록 하단 배치

## 6. 캔들차트 대신 뭘 보여줄까? (시세 카테고리)

### 결정: 캔들차트 완전 제거. "가격 비교 배틀 카드"로 대체.

### 근거
캔들차트는 트레이딩 앱의 언어다. 게임쇼에서는 "비교"가 더 직관적이다.
현재 코드(`generatePriceComparisonQuestion`)도 이미 "A vs B 누가 더 오를까?" 비교 구조다.
이걸 시각적으로 살린다.

### 시세 카테고리 전용 화면

```
+------------------------------------------+
|                                          |
|  📈 시세 예측          🍌 2,450  🏆 1,230 |
|  ─────────────────────────(금색 테마)──── |
|                                          |
|  R O U N D  7                            |
|                                          |
```

## 7. 역배 보너스 시각화

### 메커니즘
- 선택 비율이 불균등할수록 소수파에게 높은 배수 적용
- 예: A 70% / B 30% -> B 선택자에게 x1.7 배수

### 시각화 설계

**선택 후 (OPEN 상태에서)**:

```
┌──────────────────────────────────────┐
│                                      │
│  소수파 보너스                         │
│                                      │
│  🅰 ████████████████████░░░░░░ 72%   │
│  🅱 ░░░░░░░░████████░░░░░░░░░ 28%   │
│                                      │
│  ⚡ 🅱 선택 시 역배 x1.8 보너스!       │
│                                      │
│  (나의 선택: 🅱 — 역배 적용 중!)        │
│                                      │
└──────────────────────────────────────┘
```

**결과 공개 시 (역배 적중)**:

```
┌──────────────────────────────────────┐
│                                      │
│  🔥🔥🔥  역 배  적 중 ! 🔥🔥🔥        │
│                                      │
│  기본 점수: 100                       │
│  역배 보너스: x1.8                    │
│  ─────────────────                   │
│  최종 점수: 180점!                    │
│                                      │
│  "소수파의 반란! 침팬지 역전승!"        │
│                                      │
└──────────────────────────────────────┘
```

**연출**:
- 역배 적중 시: 일반 정답 연출 + 추가로 금색 폭죽 파티클 + "역배 적중!" 배너
- 사운드: "짜잔~!" + 코인 쏟아지는 효과음
- 역배 배수가 높을수록 (x2.0 이상) 연출이 화려해짐

**역배 배수 공식**:
```
minority_ratio = min(A%, B%) / 100
bonus_multiplier = 1.0 + (1.0 - minority_ratio * 2)
// 50:50 -> x1.0 (보너스 없음)
// 60:40 -> x1.2
// 70:30 -> x1.4
// 80:20 -> x1.6
// 90:10 -> x1.8
cap: x2.0
```

## 12. 롤백 계획

현재 게임 화면 컴포넌트를 바로 수정하지 않고, 새로운 `GameShowLayout` 컴포넌트를 별도로 만든 뒤 feature flag로 전환한다.

```
NEXT_PUBLIC_GAME_SHOW_UI=true  -> 새 게임쇼 UI
NEXT_PUBLIC_GAME_SHOW_UI=false -> 기존 트레이딩 UI
```

문제 발생 시 환경변수 하나로 즉시 롤백 가능.
