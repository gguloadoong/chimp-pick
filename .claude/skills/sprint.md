# 스프린트 플래닝 🦍

## 진행 절차

### 1. 스프린트 목표 설정
**김가즈아 (PM)**:
- 로드맵(`.project/roadmap.md`) 기준 이번 스프린트 목표 제시
- 백로그(`.project/backlog.md`)에서 후보 항목 선별

### 2. 기술 검토
**이떡상 (FE) + 최풀매수 (BE)**:
- 각 백로그 항목의 기술적 난이도 평가
- 스토리 포인트 산정 (1, 2, 3, 5, 8, 13)
- 기술적 블로커 식별

### 3. 디자인 의존성
**박도파민 (Designer)**:
- 디자인 준비 상태 확인
- 디자인 미완료 항목은 디자인 스프린트 별도 진행

### 4. QA 계획
**장손절 (QA)**:
- 테스트 계획 수립
- 자동화 대상 식별
- QA 일정 역산

### 5. 그로스 체크
**한물타기 (Strategist)**:
- 메트릭 영향도 확인
- A/B 테스트 필요 항목 식별

### 6. 스프린트 확정
**김가즈아 (PM)**:
- 최종 스프린트 백로그 확정
- 에이전트별 담당 항목 배분
- `.project/backlog.md` 업데이트

## 사용법
\`\`\`
/sprint          # 다음 스프린트 플래닝
/sprint 3        # Sprint 3 플래닝
\`\`\`

## 산출물
- 스프린트 백로그 (backlog.md 업데이트)
- 에이전트별 담당 항목
- 스프린트 목표 요약
