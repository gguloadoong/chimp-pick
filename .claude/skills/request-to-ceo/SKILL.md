# CEO에게 의견 요청 📋

## 개요
에이전트가 자체 판단으로 결정하기 어려운 사안을 CEO(사용자)에게 에스컬레이션합니다.
명확한 선택지와 각 선택의 트레이드오프를 함께 제시합니다.

## 에스컬레이션 기준
다음 경우에 CEO에게 요청합니다:
1. **스코프 변경**: MVP 범위를 넘어서는 기능 추가/삭제
2. **예산 영향**: 인프라 비용, 외부 서비스 결제 필요
3. **일정 변경**: 스프린트 목표 조정, 런칭일 변경
4. **팀 갈등**: 에이전트 간 의견 합의 불가
5. **규제/법률**: 도박법, 개인정보보호 등 법적 판단
6. **외부 의존**: 파트너십, API 계약 등

- 결정사항을 `.project/decisions.md`에 기록
- 관련 에이전트에게 결정 전달
- 필요 시 백로그/로드맵 업데이트
